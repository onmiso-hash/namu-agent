# 세션 기록 — 2026-06-24

## 작업 개요

Google Gemini API 어댑터를 추가하고, CLI 진입점(`main.py`)을 구현했다.
단일 명령·REPL 두 가지 실행 방식과 `.env` 기반 키 관리가 가능해졌다.

---

## 만든 것

### Gemini API 어댑터 (`adapters/gemini_api.py`)

| 항목 | 내용 |
|------|------|
| 패키지 | `google-genai` (`from google import genai`) |
| 인증 | `GEMINI_API_KEY` 환경변수 또는 `config.GEMINI_API_KEY` |
| 기본 모델 | `gemini-2.5-flash` |
| priority | `5` (하드오버라이드 — API 어댑터 중 최저) |
| 클라이언트 | 지연 초기화 (`_get_client()`) — 키 없이도 인스턴스 생성 가능 |

구현 메서드:
- `is_available()` — 환경변수 또는 config 키 존재 여부 확인
- `generate()` — `types.GenerateContentConfig`로 단일 프롬프트 실행
- `chat()` — `Role.SYSTEM` 분리 → `system_instruction`, 나머지를 `types.Content` 리스트로 변환
- `_build_result()` — 토큰 집계 공통 헬퍼 (`usage_metadata` 파싱)

Gemini 역할명 차이 처리:
```python
role="model" if m.role == Role.ASSISTANT else "user"
```

### CLI 진입점 (`main.py`)

두 가지 실행 모드:
```bash
python main.py "할 일"   # 단일 명령 — 승인 게이트 → 응답 출력
python main.py           # REPL 모드 — 멀티턴 히스토리 유지
```

옵션:
| 플래그 | 기본값 | 설명 |
|--------|--------|------|
| `--no-approve` | off | 승인 게이트 비활성화 |
| `--system PROMPT` | 없음 | 시스템 프롬프트 주입 |

REPL 동작:
- 입력 루프마다 `list[Message]` 히스토리에 누적 → `orc.run(messages=history)` 전달
- 승인 취소 시 마지막 메시지를 히스토리에서 제거해 대화 맥락 보존
- `quit` / `exit` / Ctrl+C / Ctrl+D 로 종료

`.env` 로드:
```python
load_dotenv(Path(__file__).parent / ".env")   # import 전 최상단
```

## 주요 설계 결정

### priority 오버라이드
`base.py`의 `priority`는 `AdapterType`에서 자동 계산 (API=3).
Gemini는 폴백용이므로 `@property priority`를 오버라이드해 `5`를 반환.
다른 API 어댑터들과의 충돌 없이 독립적 우선순위 설정 가능.

### 지연 초기화 (lazy init)
GPTAPIAdapter와 동일한 패턴 채택.
`__init__`에서 클라이언트를 생성하지 않아 키 없이도 어댑터 목록에 올릴 수 있고,
`is_available()` false 반환으로 자연스럽게 스킵된다.

### REPL 히스토리 관리
취소된 요청 → `history.pop()`으로 복구.
오케스트레이터가 `messages` 인자를 받으면 `adapter.chat()`을 호출하는 기존 구조를 그대로 활용.

## 버그 수정

### 1. `UnicodeEncodeError` — 터미널 입력 깨진 문자 처리 (`core/orchestrator.py`)

**증상:** 특수문자가 포함된 입력으로 오류 발생 시, 실패 기록을 파일로 쓰는 과정에서
`UnicodeEncodeError: 'utf-8' codec can't encode characters: surrogates not allowed`가 발생해 프로그램 전체가 크래시.

**원인:** `Path.write_text(encoding="utf-8")`는 surrogate 문자를 허용하지 않음.
에러가 발생하면 오케스트레이터가 실패 내용을 `task.md`에 기록하려다 재차 예외 발생.

**수정:** `_write_task_files`와 `_append_learning`의 모든 `read_text` / `write_text`에 `errors="replace"` 추가.
인코딩 불가 문자는 `?`로 대체하고 정상 진행.

### 2. `aclose(): asynchronous generator is already running` (`adapters/claude_subscription.py`)

**증상:** SDK가 에러 `ResultMessage`를 반환하면 `raise RuntimeError`로 탈출 → Python이 async generator 정리 시도 →
이미 실행 중인 generator를 닫으려다 두 번째 예외 발생 → 연쇄 traceback.

**수정:** `async for` 루프를 `try/finally`로 감싸고 `await gen.aclose()` 명시적 호출.
에러 메시지는 `error_msg` 변수에 담아두고 루프 종료 후 `raise`.

```python
gen = query(prompt=prompt, options=options)
try:
    async for message in gen:
        ...
        elif isinstance(message, ResultMessage):
            if message.is_error:
                error_msg = ...
                break
finally:
    await gen.aclose()

if error_msg:
    raise RuntimeError(f"claude-agent-sdk 오류: {error_msg}")
```
