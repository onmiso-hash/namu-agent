# namu-30-memory-sync — 설치형(~/.namu) 교훈 자동 백업·멀티 PC 동기화
📅 생성 2026-07-11 [hp] · 🔗 관련: namu-26(저장소 이원화), namu-27(세션 자기동기화), namu-29(훅 자급자족)

## 목적
설치형 사용자의 교훈(~/.namu)이 기기 로컬에 갇혀 있다 — 디스크가 죽으면
기억이 소멸하고, 멀티 PC에서는 기억이 기기마다 분열된다. NAMU의 핵심
가치(누적되는 기억)를 설치형에서도 지키려면 자동 백업·동기화가 필요하다.
방식: 풀자동 git (record 시 자동 commit+push, 세션 시작 시 자동 pull).

## 완료조건
- [x] 동기화 설정 절차 제공 — 사용자가 준비한 원격 URL을 ~/.namu에 연결하고
      동기화를 명시적으로 활성화하는 명령/절차 (git init + remote + 활성화 마커)
- [x] 동기화는 명시적 활성화 시에만 동작 — 개발 repo(clone형)가 데이터 루트인
      경우 auto-push가 오발동하지 않아야 한다 (git 저장소 존재만으로 발동 금지)
- [x] namu_record 성공 시 자동 commit+push — 실패는 무음(세션·record 차단 금지)
      + 물증 로그 남김
- [x] 세션 시작 훅에서 자동 pull — 실패 무음+물증 로그, pull 후 기존 캐시
      재생성 로직(yaml↔db 불일치 감지)과 자연 연동 확인
- [x] 동기화 미설정 환경에서 기존 동작 무손상 (record/recall/브리핑 정상)
- [x] 기존 테스트 통과 + 신규 로직 테스트 추가 (79 passed, 신규 12)
- [x] 설치형 조건(개발 repo 밖 + ~/.namu) 라이브 실측 PASS — 두 위치 간
      push→pull로 교훈이 실제 이동하는 것까지 확인 (온보딩 왕복 + 실훅 브리핑 반영)
