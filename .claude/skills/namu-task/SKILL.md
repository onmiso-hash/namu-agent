# NAMU 작업 오케스트레이션

다음 순서를 지킨다. 너(오케스트레이터)는 직접 코드를 짜지 않고 위임한다.

1. 맥락 로딩 — namu_recall로 이번 작업과 관련된 과거 교훈을 먼저 조회한다.
2. 작업 분할 — 받은 일을 코딩 워커에게 넘길 단위로 쪼갠다.
3. 워커 명단 확인 — repo 루트의 namu_workers.yaml을 읽는다. engine이 native면 Agent 도구로 해당 서브에이전트를 호출한다. (외부 엔진은 MVP 미지원이므로, native가 아니면 사용자에게 알리고 멈춘다.)
4. 코딩 위임 — namu-coder 서브에이전트에게 구현을 맡긴다.
5. 검수 위임 — namu-reviewer 서브에이전트에게 산출물 검사를 맡긴다. 검수는 pass/fail + 판정 이유를 돌려준다.
6. 검수 게이트 (자동 재실행 금지) — fail이면 멈추고 사용자에게 보여준다:
   - 검수 워커가 무엇을 기준으로 fail이라 했는지(판정 이유)
   - 사용자 선택: ① 재실행(횟수 입력) / ② 통과 처리 / ③ 중단
   - 재실행이면 입력한 N회까지 4~5단계를 돌리고, N회 소진 시 다시 게이트로.
7. 기록 — 작업이 끝나면 namu_record로 결과와 판단 이유를 저장한다. 사용자가 게이트를 통과시킨 경우 verified_by는 human.
