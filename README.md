# 🌳 NAMU Agent System

> Vendor-independent multi-agent system with shared memory and self-learning
> 📅 프로젝트 시작: 2026-06-23

---

## 🎯 프로젝트 목표

어느 하나의 AI에 종속되지 않는 멀티에이전트 시스템.
작업에 맞는 에이전트를 그때그때 정의/생성하고, 성공/실패 기록을 공유하며 스스로 학습해 발전하는 시스템.

---

## 🔧 핵심 기능 (5가지)

1. **Agent 정의/생성** — 작업에 맞는 에이전트를 그때그때 만들어내기
2. **Task 분할/배분** — 큰 작업을 쪼개서 적절한 에이전트에게 맡기기
3. **Agent 관리** — 실행 중인 에이전트들 상태 모니터링/제어
4. **기록 공유** — 성공/실패 이력을 모든 에이전트가 볼 수 있게 저장
5. **자가학습** — 쌓인 기록으로 스스로 개선 (적응형 학습)

---

## 🏛️ 설계 철학: 하네스 엔지니어링 기반

하네스 엔지니어링 철학 적용 — 에이전트가 실수했을 때 프롬프트를 수정하는 게 아니라,
구조적으로 재발하지 않도록 시스템을 개선하는 방식.

**하네스에서 가져온 것:**
- 파일이 곧 기억 (task.md / brief.md / result.md / log.md)
- 오케스트레이터-워커 구조
- 워커 호출 전 승인 게이트
- append-only 로그 (삭제 금지)
- 벤더 독립 원칙

**NAMU만의 추가 기능:**
- 수동 학습 → **자동 학습 루프** (핵심 차별점)
- 파일 + **SQLite** (패턴 분석/추출용)
- **GitHub 동기화** (다중 PC 지원)
- **AI 어댑터** (API + 구독 + 로컬 모두 지원)

---

## 🤖 AI 어댑터 구조

```
NAMU
  ↓
AI 어댑터 (연결 다리)
  ├── API 방식: Claude API / GPT API / Gemini API 등
  ├── 구독 계정: Claude.ai / ChatGPT Plus 등
  └── 로컬 모델: Ollama (Llama, Mistral 등)
```

**우선순위:** 로컬 모델 → 구독 계정 → API 호출

---

## 🧠 학습 방식

```
초기: 사람이 직접 감독 (Human-in-the-loop)
  → 판단 기록 누적
    → 패턴 분석 → 기준 자동 도출
      → AI가 자율 판단 (Human-on-the-loop)
```

> ⚠️ 핵심: 결과뿐 아니라 **판단 이유**까지 기록해야 정확한 기준 도출 가능

---

## 🔄 검증 루프

```
작업 유형 감지
  ├── 코딩 작업  → 린터 + 테스트 자동 실행
  ├── 문서 작업  → 형식/완성도 체크
  ├── 분석 작업  → 기준 충족 여부
  └── 기타 작업  → 사용자 정의 검증
```

---

## 📁 폴더 구조

```
namu-agent/
├── core/       ← 핵심 엔진 (오케스트레이터)
├── adapters/   ← AI 어댑터들
├── memory/     ← 공유 학습 기억 (GitHub 동기화)
├── tasks/      ← 작업별 기록
└── db/         ← SQLite (분석/패턴 추출)
```

---

## 🗺️ 로드맵

- **1단계 (현재):** 개인용 시스템 완성
- **2단계:** 공개 배포 + 개인 메모리 연동
- **3단계:** 공개 메모리 풀 (커뮤니티 집단지성, 선택적 기여/구독)
