셀프호스팅형 (이 문서)

Claude Code·agy 없이 웹 브라우저의 claude.ai에서 교훈을 recall·record·search 합니다.

지금 이 문서
💡MCP가 뭔지 모르고 오셨다면 — MCP(Model Context Protocol)는 AI가 외부 도구·데이터에 연결할 때 쓰는 공통 규격입니다. NAMU는 이 규격으로 "교훈 저장소"를 하나의 도구 서버로 노출하고, 웹 Claude를 포함한 어떤 MCP 클라이언트든 같은 방식으로 연결할 수 있게 합니다. 이 문서는 그 서버를 직접 인터넷에 상시로 띄워 운영하는(셀프호스팅) 절차를 다룹니다.

앞의 두 형태는 로컬에서 CC·agy가 mcp_server.py를 표준입출력(stdio)으로 직접 실행합니다(네트워크를 안 탑니다). 셀프호스팅형은 같은 기억 코어만 네트워크 너머로 노출하는 것입니다 — 그래서 아래 1절의 한계를 반드시 먼저 읽어야 합니다.

1 시작 전 알아둘 것 — 웹에서 되는 것 / 안 되는 것

셀프호스팅형은 로컬 설치형과 겉보기엔 비슷하지만, 실제로 쓸 수 있는 기능은 훨씬 좁습니다.

✅ 되는 것 — 도구 3개

namu_recall(기억 꺼내기) · namu_record(기억 저장하기) · namu_search(패턴 찾기). 로컬 설치형과 완전히 같은 기억 풀을 씁니다.

❌ 안 되는 것

기능왜 없나요
세션 자동 브리핑 · /namu-task · 서브에이전트(워커) 호출 전부 Claude Code·agy 쪽 "자동화" 기능이라 웹에는 대응 개념 자체가 없습니다. 웹 Claude가 스스로 recall을 부르게 하려면 도구 설명(description)이 유일한 유도 수단입니다.
namu_sync_setup 서버의 git 원격 저장소 주소를 재배선하는 도구입니다. 원격 호출자에게 그대로 열어두면 원격 저장소 탈취로 이어질 수 있어 웹에서는 원천 차단합니다 — 서버 쪽 동기화 배선은 운영자(자기 PC의 git 자격증명, 또는 클라우드는 배포 스크립트)가 대신합니다.
tasks(작업 상태) tasks는 "프로젝트 폴더 귀속" 개념인데, 웹 대화에는 대응하는 프로젝트 폴더 개념 자체가 없습니다.
⚠즉 셀프호스팅형으로 얻는 건 "웹 브라우저에서 교훈을 조회·검색·기록"뿐입니다. NAMU의 작업 오케스트레이션 기능은 여전히 Claude Code·agy(설치형·clone 개발형) 몫입니다.

2 인증 — 시크릿을 어떻게 확인하나

서버는 두 인증 방식을 동시 지원하며, 설정한 조합에 따라 동작이 갈립니다.

설정 조합동작
NAMU_HTTP_TOKEN만주소는 /mcp. 모든 요청에 헤더(x-api-key 또는 Authorization: Bearer <token>) 검증만 통과하면 됩니다.
NAMU_HTTP_PATH_SECRET만 (권장)주소가 /mcp/<secret>로 바뀝니다. 그 주소로만 요청하면 헤더 없이 통과합니다.
둘 다 설정AND — 시크릿 경로로 요청하고 헤더도 맞아야 통과합니다. 하나만 맞으면 실패합니다.
둘 다 없음 + NAMU_HTTP_ALLOW_NOAUTH=1무인증. 로컬 테스트 전용 — 공개 인터넷에는 절대 쓰지 마세요.
둘 다 없음(위 옵션도 없음)서버가 기동 자체를 거부합니다(무인증 공개 노출 방지).
💡권장: 시크릿 경로 단독 모드. claude.ai의 Request headers(헤더) 설정 UI는 2026-07 기준 베타·점진 롤아웃 중이라 계정에 아예 안 보일 수 있습니다. 헤더 UI가 없으면 NAMU_HTTP_PATH_SECRET만 설정하고 URL 자체(https://<host>/mcp/<secret>)로 인증하는 방식이 가장 간단합니다. 헤더 UI가 보인다면 NAMU_HTTP_TOKEN을 추가로 얹어 이중화해도 됩니다.
💡헤더 인증을 쓸 경우 서버가 실제로 검사하는 헤더 이름은 x-api-key 또는 Authorization: Bearer <token> 둘뿐입니다. 커넥터 등록 시 헤더 이름을 반드시 이 중 하나로 맞추세요.

3 연결 절차 A — 내 PC + 터널 (지금 바로 체험)

배포물이 필요 없는 가장 간단한 경로입니다. "지금 당장 체험해보고 싶다"에 최적화돼 있습니다. 대신 PC를 계속 켜둬야 하고, 터널 주소가 바뀔 때마다 커넥터를 다시 등록해야 합니다.

1

서버를 기동합니다

clone 개발형이면 repo 루트에서, 설치형이면 설치된 namu-plugin 폴더 경로에서 실행합니다.

🖥 로컬 PC 터미널 
# 시크릿 경로 단독 모드 (권장)
NAMU_HTTP_PATH_SECRET="<임의의 긴 문자열>" \
  uv run --script namu-plugin/http_server.py

기본 바인드는 127.0.0.1:8765(자기 PC 안에서만 접속 가능)입니다 — 터널 클라이언트가 같은 PC에서 이 포트에 접속하므로 그대로 충분합니다.

2

터널로 공개 HTTPS 주소를 확보합니다

claude.ai는 공개 인터넷에서 접근 가능한 URL이 필수입니다(사설망·VPN 뒤는 불가). PC를 상시 공개 서버로 열 수 없다면 터널을 씁니다 — 지금 이 PC의 로컬 서버를 외부에서도 접속 가능하게 임시로 뚫어주는 중계 통로입니다.

터널실측 결과
Cloudflare quick tunnelURL은 발급되나 API가 간헐적으로 500 — 불안정
localhost.runURL은 발급되지만 실제 포워딩 자체가 안 됨 — 실사용 불가
pinggy (권장)실제로 동작 확인됨 — 터널 경유 실호출까지 성공
⚠무료 터널 서비스 상태는 계속 바뀔 수 있습니다. 특정 터널 하나에 고정하지 말고, 아래 3단계 curl 검증을 매번 거치세요.
3

claude.ai에 등록하기 전, curl로 먼저 확인합니다

curl은 터미널에서 HTTP 요청을 직접 보내볼 수 있는 도구입니다 — 브라우저 없이 서버가 살아있는지 확인할 때 씁니다.

🖥 로컬 PC 터미널 (터널 URL을 향해 요청) 
curl -sS -X POST "https://<터널 URL>/mcp/<secret>" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-06-18","capabilities":{},"clientInfo":{"name":"smoke","version":"0"}}}'
✓정상 JSON-RPC 응답(result 포함)이 오면 등록 단계로 넘어가세요. 타임아웃·연결거부·502류 응답이면 "URL은 있는데 안 뚫린 것"이니 다른 터널로 바꾸세요.

터널이 도메인을 발급하면 NAMU_HTTP_ALLOWED_HOSTS도 함께 설정해야 합니다(안 하면 421 에러).

4

claude.ai에 Custom Connector를 등록합니다

claude.ai → 설정 → Connectors → Add custom connector. Remote MCP server URL에 시크릿 경로 단독 모드라면 https://<터널 URL>/mcp/<secret>을 넣습니다. 헤더 인증을 함께 쓴다면 URL은 /mcp(또는 /mcp/<secret>)로 두고 Request headers에 x-api-key: <토큰>을 추가합니다.

등록 후 대화에서 "namu 메모리에서 recall 해봐"처럼 직접 도구 호출을 유도해 동작을 확인하세요 — 자동 브리핑이 없으므로 첫 호출은 직접 유도해야 합니다.

⚠PC가 꺼지거나 서버 프로세스가 죽으면 웹에서 접속이 끊깁니다. 상시 가동이 필요하면 아래 5절(절차 B)로 가세요.

4 연결 절차 B — 상시 서버 (클라우드 컨테이너)

"한 번 등록해두고 계속 쓰고 싶다"에 최적화된 경로입니다. PC를 껐다 켜도 죽지 않는 상시 서버(사용자 소유 미니PC·VPS 등)와 고정 도메인이 있다는 전제입니다. 고정 도메인이면 claude.ai 커넥터 등록도 최초 1회로 끝납니다.

✓검증 상태 — 2026-07-17, 사용자 소유 상시 서버에 실제 배포해 전 구간(빌드 → 고정 터널 → claude.ai 커넥터 등록 → 웹에서 recall/search/record 실호출 → 다른 PC에서도 그 기록이 조회됨)을 실측 검증했습니다. 다만 Railway·Fly.io 같은 PaaS로 배포하는 경로는 아직 검증된 적이 없습니다.

4-1. 사전 준비

교훈 저장소로 쓸 비공개 GitHub 저장소(설치형에서 쓰던 저장소를 재사용해도 됩니다) — 여기에 push 권한이 있는 PAT(개인 액세스 토큰, 비밀번호처럼 절대 노출하면 안 됩니다)가 필요합니다.

https://x-access-token:<PAT>@github.com/<user>/<repo>.git

4-2. 시크릿을 어디에 둘까 — 계층화

이 컨테이너를 기동하려면 최소 두 값(NAMU_SYNC_REMOTE, 그리고 NAMU_HTTP_PATH_SECRET 또는 NAMU_HTTP_TOKEN)이 필요합니다. 이런 값을 시크릿이라 부릅니다 — 외부에 노출되면 안 되는 값입니다.

기본 경로 (대부분에게 권장)

서버 로컬 .env 파일 / 플랫폼 환경변수 화면

전제 조건이 가장 적고, 시크릿이 필요한 서버 한 곳에만 존재해 노출 표면이 좁습니다. 별도 CI 구축이 필요 없습니다.

부록 (고급 — 이미 CI/CD가 있는 사람만)

CI 시크릿 저장소(GitHub Actions Secrets 등)

push하면 자동으로 서버까지 배포되는 파이프라인을 이미 운영 중일 때만 의미가 있습니다. 없다면 왼쪽 기본 경로로 충분합니다.

🖧 서버, docker-compose.yml과 같은 폴더 
# docker-compose.yml
services:
  namu-remote-mcp:
    image: namu-remote-mcp:latest   # 빌드는 4-3에서 미리 해둡니다
    env_file:
      - .env                        # 시크릿 2종은 여기서 읽어옵니다
    environment:
      NAMU_HTTP_ALLOWED_HOSTS: "mcp.example.com"
    ports:
      - "8765:8765"
    volumes:
      - namu-data:/root/.namu       # 컨테이너를 지워도 교훈 데이터가 남도록
    restart: unless-stopped

volumes:
  namu-data:
🖧 서버, .env (반드시 git에 커밋하지 말 것) 
NAMU_SYNC_REMOTE=https://x-access-token:<PAT>@github.com/<user>/<repo>.git
NAMU_HTTP_PATH_SECRET=<임의의 긴 문자열>
⚠.env가 조용히 사라지는 사고 — 일반 원칙. 배포 자동화가 "원본 .env → 서비스별 폴더에 사본 복사" 구조라면, 사본을 직접 고쳐도 다음 배포에서 원본으로 덮어써져 수정 사항이 소리 없이 지워질 수 있습니다. .env를 고치기 전에 "이게 원본인지, 매 배포마다 재생성되는 사본인지"부터 확인하세요.

4-3. 이미지 빌드 · 기동

이미지는 서버 실행에 필요한 모든 것을 굳혀둔 꾸러미, 컨테이너는 그 이미지를 실제로 띄운 상태입니다. 빌드는 docker build로 따로 하고, compose는 완성된 이미지 태그만 참조하는 패턴을 권장합니다.

⛔Windows 함정 — Windows Docker Desktop의 docker compose는 빌드 소스 위치에 GitHub URL을 직접 넣으면 로컬 파일 경로로 잘못 해석해 빌드가 실패합니다. 그래서 compose 파일 안에는 GitHub URL을 넣지 말고, 빌드는 아래처럼 별도 docker build 명령으로 분리하세요.
🖧 서버 터미널 
docker build -f deploy/Dockerfile -t namu-remote-mcp:latest \
  'https://github.com/onmiso-hash/namu-agent.git#v0.1.25'

PowerShell(Windows 서버)에서 실행한다면 반드시 URL을 작은따옴표로 감싸세요 — 안 그러면 #v0.1.25가 주석 시작 기호로 오인돼 잘려나갑니다.

🖧 서버 터미널 (docker-compose.yml·.env가 있는 폴더) 
docker compose up -d

컨테이너 기동 시 자동으로 처리되는 것: NAMU_SYNC_REMOTE가 없으면 즉시 기동 거부 → ~/.namu가 없으면 clone, 있으면 pull → git identity가 비어 있으면 기본값으로 채움 → git remote를 자동 배선 → http_server.py 기동. 각 단계는 실패 시 조용히 넘어가지 않고 즉시 종료합니다.

4-4. 알려진 캐비엇

  • $PORT 자동 브리징 없음 — Railway 등 일부 PaaS는 컨테이너가 $PORT 환경변수로 리슨하길 기대합니다. 이 이미지는 NAMU_HTTP_PORT 고정 기본값(8765)이라 자동으로 따라가지 않습니다. 해당 플랫폼이면 NAMU_HTTP_PORT를 직접 override해야 합니다.
  • PaaS 배포 자체가 아직 검증 밖입니다 — 자기 소유 서버에 compose로 올리는 경로는 실측 완료됐지만, Railway·Fly.io 같은 플랫폼 배포는 이 문서만으로 충분한지 확인된 바 없습니다.

5 A vs B, 뭘 고를까?

🚀 절차 A — 내 PC + 터널

지금 바로 체험해보고 싶다면

자기 소유 도메인·상시 서버가 없어도 몇 분 안에 웹 연결까지 끝낼 수 있습니다. 대신 PC를 계속 켜둬야 하고, 터널 주소가 바뀔 때마다 커넥터를 다시 등록해야 합니다.

🏠 절차 B — 상시 서버(Docker)

자기 소유 상시 서버·고정 도메인이 있다면

6 NAMU_HTTP_ALLOWED_HOSTS — 421 에러 해소

터널이나 리버스 프록시를 거쳐 접속하면 421 Misdirected Request가 날 수 있습니다.

원인 — 서버가 DNS rebinding(공격자가 도메인을 악용해 브라우저가 사설 네트워크로 요청을 보내게 속이는 공격) 방지를 자동으로 켜고, 허용 Host 헤더를 로컬호스트로만 고정하기 때문입니다. 터널·프록시 도메인의 Host 헤더는 이 목록에 없어 거부됩니다.

해법 — 터널·프록시 도메인을 쉼표로 구분해 추가합니다(로컬호스트 허용은 그대로 유지된 채 확장됩니다):

🖥 로컬 PC 터미널 
NAMU_HTTP_ALLOWED_HOSTS="my-tunnel-1.example.com,my-tunnel-2.example.com" \
  uv run --script namu-plugin/http_server.py
💡*를 주면 이 보호 자체를 비활성화합니다(도메인이 배포마다 유동적일 때의 명시적 opt-out). 컨테이너 이미지는 기본값이 이미 *입니다. 매번 도메인을 알 수 있는 환경이라면 실제 도메인을 지정하는 편이 더 안전합니다.

7 보안 주의

⛔uvicorn(내부 웹 서버) 액세스 로그에 시크릿 경로가 평문으로 남습니다. NAMU_HTTP_PATH_SECRET을 URL 경로로 쓰는 방식의 구조적 특성입니다 — 로그를 공유·노출하면 그 즉시 시크릿이 유출된 것과 같습니다. 유출이 의심되면 지체 없이 값을 교체(서버 재기동)하세요.
⚠터널을 끝낼 때 시크릿도 함께 폐기(교체)하는 것을 권장합니다 — 터널 URL은 만료돼도, 같은 시크릿을 다음 세션에 재사용하면 과거 로그에 남은 값이 여전히 유효합니다.
💡이 서버는 HTTPS를 직접 종단하지 않습니다 — 터널(cloudflared·pinggy 등)이나 플랫폼(Railway 등)이 HTTPS를 담당한다는 전제입니다. HTTP 평문 구간이 남지 않도록 반드시 HTTPS를 종단하는 터널·플랫폼을 쓰세요.

환경변수기본값의미
NAMU_HTTP_PATH_SECRET미설정시크릿 URL 경로. 설정 시 주소가 /mcp/<secret>로 바뀝니다(권장 방식)
NAMU_HTTP_TOKEN미설정헤더 인증 토큰(x-api-key 또는 Authorization: Bearer)
NAMU_HTTP_ALLOW_NOAUTHoff1이면 무인증 기동 허용. 로컬 테스트 전용, 공개 노출 금지
NAMU_HTTP_HOST127.0.0.1바인드 호스트. 컨테이너는 0.0.0.0으로 override됨
NAMU_HTTP_PORT8765바인드 포트
NAMU_HTTP_ALLOWED_HOSTS로컬호스트만421 해소용 허용 Host 목록. *는 비활성화. 컨테이너 기본값은 *
NAMU_SYNC_REMOTE없음(컨테이너 전용)클라우드 컨테이너가 ~/.namu를 clone/pull할 git remote URL. 미설정이면 기동 거부
NAMU_MACHINE호스트명기록된 교훈의 machine 필드. 컨테이너 기본값은 web
