## 1. 목표

클로드코드와 agy **둘 다**, 세션을 새로 켜면 같은 내용을 자동으로 띄운다:

> 📌 진행 중: ○○ 작업 / 다음 할 일: △△ / 최근 로그 / 💡 관련 교훈(예전에 □□ 실패했으니 조심)

**핵심: 내용은 100% 같고, 출력 봉투만 호스트별로 다르다.**

### ⚠️ 왜 이 작업이 생겼나 (배경 — 꼭 이해하고 갈 것)
- 현재 `session_recall.py`(클로드코드 자동주입)는 **교훈만**(`db.recall(limit=5)`) 띄운다. **작업 상태(tasks/)는 안 띄운다.**
- 이유: `session_recall.py`는 **06-25**에 만들어졌는데, 작업 상태 층(`context.md` 3파일 구조)은 **06-26**에 생겼다. 만들 때 상태 개념이 없어서 빠진 것 — 의도가 아니라 타이밍 사고.
- → 이번에 **양쪽을 같이** 올려 "교훈 + 작업 상태"를 자동주입하게 통일한다. ("두 수정은 동시에, 나중에 하면 까먹는다" 원칙)

## 2. 핵심 설계 — 공유 헬퍼 + 호스트별 래퍼

```
            [ namu-plugin/session_context.py ]   ← 신규 공유 헬퍼 (뇌)
            활성 task 찾기 + 교훈 조회 + 마크다운 문자열 조립
                  ▲                         ▲
        호출 │ build_context_markdown(conn, machine)
                  │                         │
   session_recall.py (클로드코드)   session_inject.py (agy)
   래퍼: SessionStart 출력          래퍼: ephemeralMessage + 플래그
```

- **헬퍼(C층, 포터블)**가 "무슨 내용을 띄울지"를 단독 결정 → 두 호스트가 동일 함수 호출 → **내용 어긋남 원천 차단**.
- 두 훅은 "그 문자열을 자기 호스트 포맷으로 감싸기"만 한다.
- 헬퍼는 `conn`을 인자로 받는다(읽기 계열 철학과 일관, `:memory:` 주입으로 단위 테스트 가능).

> ⚠️ **분리 원칙 (엮지 말 것):** 세션 1회 **플래그는 자동주입 훅에만** 적용된다. 사용자가 직접 호출하는 **MCP 도구(`namu_recall`/`namu_search`)는 플래그와 완전 무관**하게 언제나 동작해야 한다. 플래그 파일을 검사하는 코드는 `session_inject.py` 안에만 존재할 것 — `mcp_server.py`/`db.py`/`session_context.py`는 플래그의 존재조차 몰라야 한다. (자동주입 1회 ≠ 수동 조회 제한)

## 4. ⚠️ 확정 사실 (agy 측, 검증 완료 — 바꾸지 말 것)

이전 세션에서 공식 docs + HP 스모크 실측으로 닫은 것들. 재실측은 환영, 임의 변경 금지.

1. **agy 출력 형식**
   - 주입: stdout에 정확히 → `{"injectSteps":[{"ephemeralMessage":"<문자열 하나>"}]}`
   - 미주입: stdout에 → `{}`
   - `injectSteps` 원소는 `toolCall`/`userMessage`/`ephemeralMessage` 중 **하나만**. 우리는 **`ephemeralMessage`**(시스템 컨텍스트).
2. **세션 1회 = `conversationId` + 플래그 파일 존재 여부**
   - stdin(JSON)에서 `conversationId` 읽기. (`invocationNum`은 매 turn 0 리셋이라 폐기, `initialNumSteps`도 예측불가라 부적합. `conversationId`만 세션단위 고정.)
   - 플래그: `Path(tempfile.gettempdir()) / f"namu_injected_{conversationId}"` (WSL=/tmp·Win=%TEMP% 자동분기)
   - **판정 = 값 비교 아님, "플래그 파일 있냐 없냐".** 있으면 `print("{}")`+exit / 없으면 주입 후 `flag.touch()`.
3. **hooks.json = 네임스페이스 래핑** (`.agents/hooks.json`)
   - `{"namu-inject":{"PreInvocation":[{"type":"command","command":"<절대경로>"}]}}`
   - PreInvocation은 matcher 무시. 핸들러 필드: `type`(command만)/`command`(필수)/`timeout`(기본 30).
4. **훅 command 절대경로 필수** — agy 훅은 cwd 보장 안 함. 상대경로면 **로그 없이 조용히 죽음.** realpath로 박을 것.

## 5. 상세 스펙

### 5-1. `namu-plugin/session_context.py` (신규 공유 헬퍼)

#### 함수 ①: `find_active_task(machine) -> Path | None`  (옵션 A, 2026-06-27 확정)
```
1. tasks 루트(config 경유) 아래 tasks/*/context.<machine>.md 글로빙
2. mtime 내림차순 정렬
3. 각 파일에서 "## ▶ 다음" 섹션 본문을 추출
   (헤더 다음 줄부터 다음 "##" 또는 EOF까지, strip)
   - 본문이 "(완료)" 면 → 완료된 task, 스킵
   - 아니면 → 이게 활성 task → 그 폴더(Path) 반환
4. 끝까지 활성 없으면 → None 반환
```
- **머신별 context를 보는 게 핵심**: pull로 들어온 다른 PC의 `context.<other>.md`는 mtime이 갱신돼도 내 판정에 안 낀다(#14 "같은-초 pull" 함정 구조적 회피).

#### 함수 ②: `build_context_markdown(conn, machine) -> str | None`
```
active = find_active_task(machine)

if active:
    - task.md → 제목/목적 1~2줄 요약
    - context.<machine>.md → "## ▶ 다음" 본문 (= 다음 할 일)
    - log.md → 마지막 3~5줄 요약
    - learnings = db.recall(conn, query=<task 제목/키워드>, limit=작게)   # 옵션 A 폴백
else:
    - learnings = db.recall(conn, limit=5)                                # query 없이 최신

→ 아래 §5-1-③ 포맷으로 한 덩어리 마크다운 조립
→ 맨 끝에 저장 안내 한 줄 동봉
→ 내용이 실제로 비면(task 없고 learnings도 0) None 반환, 아니면 문자열 반환
```
- `conn`은 인자로 받는다(호출자가 열고 닫음). 헬퍼 안에서 새 conn 열지 말 것.
- 예외는 헬퍼가 삼키지 말고 **호출자(훅)가 처리**한다(훅이 exit 0 책임). 단 "활성 task 파싱 중 개별 파일 깨짐"은 헬퍼가 그 파일만 스킵하고 진행.

#### ③ 주입 마크다운 포맷 (양쪽 호스트 공통 — 여기서 단 한 번 정의)
```markdown
## 🌳 NAMU — 세션 컨텍스트 자동 로딩

### 📌 진행 중: <task 제목>
- **다음 할 일:** <context의 ▶ 다음 본문>
- **최근 로그:** <log.md 마지막 3~5줄 요약>

### 💡 관련 교훈
- [<outcome>] <task>: <reason>
- [<outcome>] <task>: <reason>

---
※ 새 교훈이 생기면 namu_record 도구로 저장하세요.
```
- 활성 task 없으면 "📌 진행 중" 블록 생략, "💡 최근 교훈"만.

#### 단위 테스트 (가벼운 비파괴)
- 임시 tasks 디렉터리 2~3개 + `:memory:` db로:
  - 완료/진행 섞였을 때 **가장 최근 진행 task**가 뽑히는지
  - 전부 완료면 None(→ learnings-only 분기) 인지
  - 조립 문자열에 "다음 할 일"과 교훈이 둘 다 들어가는지
- 기존 `test_*.py` 컨벤션에 맞춰 추가.

### 5-2. `session_recall.py` 수정 (클로드코드 / SessionStart)

- **변경점은 딱 하나**: 기존 `db.recall(limit=5)` 호출부를 → `build_context_markdown(conn, machine)` 호출로 교체.
- 나머지는 그대로 유지:
  - SessionStart 출력 메커니즘(stdout/`additionalContext`) **변경 금지** — 봉투는 그대로, 내용물만 교체.
  - `_ensure_db` + conn 열고 닫기 패턴 유지.
  - 어떤 예외든 `exit 0`(세션 시작 안 막음) 유지.
- `machine`은 config(`NAMU_MACHINE`)에서.
- 헬퍼가 `None` 반환하면(아무것도 없음) → 기존처럼 주입 없이 정상 종료.
- ⚠️ SessionStart는 천성적으로 세션당 1회라 **플래그 불필요**(agy와의 유일한 비대칭은 이 지점뿐).

### 5-3. `session_inject.py` 신규 (agy / PreInvocation)

> 참고: `session_recall.py`의 코어 임포트/`_ensure_db` 패턴을 따른다. 단 이 파일은 `namu-plugin/` **밖**(`.agents/hooks/`)이라 코어 경로를 sys.path에 추가해야 함 — `session_recall.py`가 푸는 방식과 **일관되게** 맞출 것(실제 상대 깊이는 트리 보고 확인).

```
1) PEP 723 + uv run --script (mcp_server.py 방식). 필요한 의존성만 핀.
2) stdin JSON → conversationId. 못 읽으면 print("{}") + exit 0.
3) flag = tempfile.gettempdir()/f"namu_injected_{conversationId}"
   - flag.exists() → print("{}") + exit 0   # 이 세션 이미 주입
4) _ensure_db, conn 열기
5) md = build_context_markdown(conn, machine)   # ← 클로드코드와 동일 헬퍼
6) if md:
       print(json.dumps({"injectSteps":[{"ephemeralMessage": md}]}, ensure_ascii=False))
       flag.touch()
   else:
       print("{}")
7) conn 닫기, exit 0
8) 어떤 예외든 → print("{}") + exit 0 (세션 안 막음)
```

#### 진단(임시) — 라이브 검증용, 통과 후 제거
- 발화: `echo fired >> <abs>/agy_fired.log`
- 에러: stderr `2>> <abs>/agy_py_err.log`
- 출력: stdout 로그 분리 확인

### 5-4. `.agents/hooks.json`

§4-3 포맷. command는 절대경로 uv 실행:

```json
  }
}
```
- 등록 후 **agy 재시작** 필요(훅은 세션 시작 시 스냅샷).

## 6. 라이브 검증 (양쪽 다 — 1:1 확인이 핵심)

**진행 중 task가 하나 있는 상태**를 만들어 두고:

### 클로드코드
1. 새 세션 시작 → 첫 컨텍스트에 "📌 진행 중 + 💡 교훈"이 자동으로 떠야 함 (**이제 교훈만이 아님**).
2. 작업 상태 없을 때(전부 완료): "💡 최근 교훈"만 떠야 함.

### agy
3. 재시작 → 새 세션 → 같은 "📌 진행 중 + 💡 교훈" 쪽지 자동.
4. 같은 세션 두 번째 invocation → **다시 안 뜸**(플래그).
5. 새 세션 → 다시 뜸(conversationId 갱신).
6. (음성 실패 대비) §5-3 진단 로그로 발화/에러/출력 확인.

### 1:1 대조
7. 같은 task 상태에서 **클로드코드 출력 내용 ≈ agy 출력 내용**인지 눈으로 대조(봉투만 다르고 본문 같아야 함).

검증 결과(통과/실패 + 화면) 사용자에게 보고 → 사용자가 diff 확인 후 직접 커밋.
