A Claude Code에 설치

GitHub에서 바로 플러그인으로 설치합니다 — 공개 저장소라 로그인·초대·수동 clone 없이 끝납니다.

1

플러그인을 설치합니다.

🖥 터미널에서
claude plugin marketplace add onmiso-hash/namu-agent
claude plugin install namu@namu-marketplace

💡Claude Code 대화창 안에서 하려면 /plugin marketplace add onmiso-hash/namu-agent도 똑같이 동작합니다.

2

Claude Code를 껐다 켭니다.

완전히 재시작한 뒤 새 대화 세션을 엽니다. (재시작해야 반영됩니다.)

3

제대로 됐는지 확인합니다.

💬 대화창에서
/mcp
✓성공! 목록에 plugin:namu:namu-memory · ✔ connected · 3 tools가 보이면 메모리 코어가 살아 있는 겁니다.

💡첫 세션은 아무 안내도 안 뜨는 게 정상입니다(아직 저장된 기억이 0건이라서). 설치 성공 여부는 위 /mcp로만 판단하세요.

B agy(Antigravity)에 설치

GitHub 저장소 주소를 그대로 넘기면 agy가 clone·설치를 대신 처리합니다.

1

플러그인을 설치합니다.

🖥 터미널에서
agy plugin install https://github.com/onmiso-hash/namu-agent.git
2

agy에서 새 대화 세션을 열고 확인합니다.

대화창에 /mcp를 쳐서 namu-memory가 연결돼 있으면 성공입니다.

⚠설치 직후 첫 세션에만 /mcp가 이상하게 보일 수 있습니다. 정상입니다 — 잠시 후 아래 업데이트 섹션의 /namu:update를 한 번 부르면 경로 자동 교정까지 바로 처리됩니다.

💡이미 이 repo를 clone해 둔 개발자라면 로컬 namu-plugin 폴더 경로로 agy plugin install <경로>도 그대로 동작합니다 — 일반 사용자는 위 GitHub 방식이면 충분합니다.

+ 화면 하단 상태줄 (선택)

[Namu 버전] [모델] 폴더 | 📌 진행 중 작업 | 컨텍스트% · 5h(남은시간) · 7d(남은시간) 한 줄을 보고 싶다면.

1

한 번 호출하면 자동 연결됩니다.

💬 대화창에서
/namu:statusline-setup

NAMU가 설치된 호스트(Claude Code·agy)를 자동으로 감지해 한 번에 연결합니다. 성공 메시지가 뜨면 해당 엔진을 재시작해야 화면에 반영됩니다. 안 보이면 이 명령을 다시 부르면 됩니다.

↻ 업데이트 — 새 버전 반영하기

엔진 구분 없이, 대화창에서 명령 하나면 끝납니다.

💬 대화창에서
/namu:update

설치된 엔진(Claude Code·agy)을 자동으로 찾아 각각 최신 버전으로 갱신하고, 끝나면 상태줄(statusLine) 경로까지 자동으로 다시 연결합니다. 둘 다 설치돼 있으면 둘 다, 하나만 있으면 그것만 처리합니다.

✓완료 메시지가 뜨면 갱신된 엔진을 재시작하세요. /namu:statusline-setup을 따로 다시 부를 필요가 없습니다.
💡스킬을 못 쓰는 환경이라면 — Claude Code는 터미널에서 claude plugin update namu@namu-marketplace(반드시 이름@마켓플레이스 전체 이름으로), agy는 agy plugin uninstall namu 후 agy plugin install https://github.com/onmiso-hash/namu-agent.git을 수동으로 실행한 뒤, 상태줄이 안 보이면 /namu:statusline-setup을 따로 불러 주세요.
