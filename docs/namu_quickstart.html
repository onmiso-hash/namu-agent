A Claude Code에 설치

GitHub에서 바로 플러그인으로 설치합니다 — 공개 저장소라 로그인·초대·수동 clone 없이 끝납니다.

1

플러그인을 설치합니다.

🖥 터미널에서
claude plugin marketplace add onmiso-hash/namu-agent
claude plugin install namu@namu-marketplace

💡Claude Code 대화창 안에서 하려면 /plugin marketplace add onmiso-hash/namu-agent도 똑같이 동작합니다.

2

Claude Code를 껐다 켭니다.

완전히 재시작한 뒤 새 대화 세션을 엽니다. (재시작해야 반영됩니다.)

3

제대로 됐는지 확인합니다.

💬 대화창에서
/mcp
✓성공! 목록에 plugin:namu:namu-memory · ✔ connected · 3 tools가 보이면 메모리 코어가 살아 있는 겁니다.

💡첫 세션은 아무 안내도 안 뜨는 게 정상입니다(아직 저장된 기억이 0건이라서). 설치 성공 여부는 위 /mcp로만 판단하세요.

B agy(Antigravity)에 설치

agy는 아직 GitHub 원격 설치가 없어, 소스를 먼저 내려받은 뒤(clone) 그 namu-plugin 폴더를 플러그인으로 설치합니다. clone은 설치 방식이 아니라 소스를 가져오는 준비 단계입니다.

1

저장소를 내려받습니다.

🖥 터미널에서
git clone https://github.com/onmiso-hash/namu-agent.git
2

플러그인을 설치합니다.

🖥 터미널에서
agy plugin install namu-agent/namu-plugin

경로는 방금 내려받은 폴더 안의 namu-plugin입니다.

3

agy에서 새 대화 세션을 열고 확인합니다.

대화창에 /mcp를 쳐서 namu-memory가 연결돼 있으면 성공입니다.

⚠설치 직후 첫 세션에만 /mcp가 이상하게 보일 수 있습니다. 그럴 때 아래를 한 번 실행하면(청소 후 재설치 + 경로 자동 교정) 해결됩니다.
🖥 터미널에서
namu-agent/namu-plugin/scripts/agy_reinstall.ps1

+ 화면 하단 상태줄 (선택)

[모델] 폴더 | 📌 진행 중 작업 | 컨텍스트% 한 줄을 보고 싶다면.

1

한 번 호출하면 자동 연결됩니다.

💬 대화창에서
/namu:statusline-setup

NAMU가 설치된 호스트(Claude Code·agy)를 자동으로 감지해 한 번에 연결합니다. 성공 메시지가 뜨면 해당 엔진을 재시작해야 화면에 반영됩니다. 안 보이면 이 명령을 다시 부르면 됩니다.

↻ 업데이트 — 새 버전 반영하기

엔진에 따라 방법이 다릅니다. 자기 것 하나만 보면 됩니다.

AClaude Code
🖥 터미널에서
claude plugin update namu@namu-marketplace

그다음 Claude Code를 재시작하면 끝.

⚠반드시 namu@namu-marketplace 전체 이름으로 치세요. 짧게 update namu만 쓰면 “not found”로 실패합니다.
Bagy

먼저 내려받은 폴더에서 git pull로 소스를 최신화한 뒤, 재설치 스크립트를 실행합니다.

🖥 터미널에서 (내려받은 namu-agent 폴더에서)
git -C namu-agent pull
namu-agent/namu-plugin/scripts/agy_reinstall.ps1

재설치 스크립트가 제거 → 재설치 → 경로 교정을 한 번에 처리합니다.

💡상태줄(statusLine)을 쓰고 있다면, 업데이트 뒤 /namu:statusline-setup을 한 번 더 부르세요. 버전이 바뀌면 상태줄이 조용히 사라질 수 있는데, 이 한 번으로 새 경로에 다시 연결됩니다.

데이터(교훈·작업 기록)는 설치 방식과 무관하게 항상 ~/.namu 폴더에 쌓입니다. 여러 PC에서 기억을 공유하고 싶다면 설치 후 AI에게 “namu_sync_setup을 내 비공개 저장소 URL로 실행해줘”라고 부탁하세요(선택 기능).
